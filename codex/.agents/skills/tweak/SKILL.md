# 軽微修正（tweak）

軽微修正を以下の手順で実施する。

1. **軽微判定**: AGENTS.md「軽微修正（tweak）」の条件をすべて満たすか確認する。
   - 振る舞い・ロジックの変更がない / 外部IFに触れない /
     テストの修正が不要 / 変更が小さい（数ファイル・数十行以内）
   - 1つでも外れる、または判断に迷う場合は**修正せず停止**し、理由を説明して
     通常フロー（plan-new スキル type: fix）を提案する。
   - 要件・仕様に影響すると判断した場合も同様に停止して指摘する
     （要件定義書の変更を要する修正はユーザーが明言する運用）。
2. 修正を実施する。依頼された範囲以外に手を広げない。
3. 既存テストを実行し、全パスを確認する（落ちたら修正を見直す。テスト側は触らない）。
4. `docs/TWEAKS.md` に1行追記する:
   `| 日付 | 世代 | 内容 | 理由 | 変更ファイル | 関連チケット等 |`
   （世代は要件定義書の `generation:`。関連チケットは該当があれば）
5. 変更内容・テスト結果を簡潔に報告する。同じ箇所への tweak が過去にもある場合は、
   チケット化（通常フロー）を提案する。
