# 要件定義書の作成と壁打ち

以下の手順で要件定義書を作成する。**設計や実装には進まないこと。**

1. `docs/requirements/_TEMPLATE.md` を元に `docs/requirements/requirements.md` を作成する
   （既に存在する場合は上書きせず、壁打ちモード＝手順3から始める）。
2. 依頼内容から背景・目的、機能要件（REQ-F-NN）、非機能要件（REQ-N-NN）、対象外を埋める。
   判断できない点・曖昧な点は **未決事項** に列挙する。
3. **壁打ちモード**: 未決事項や要件の穴について、1回に1〜3個ずつ質問して対話的に詰める。
   - 単に聞くだけでなく、選択肢とトレードオフを提示して議論する。
   - 決まったことは要件に反映し、**変更履歴に「何をどう決めたか・なぜか」を追記する。**
   - 未決事項が解消されるまで繰り返す。
4. 未決事項がなくなったら全体を提示し、承認（status: approved）を求めて停止する。
   承認後の次ステップは plan-new スキル（実装計画書）。
