# 詳細設計書の作成（任意層）

詳細設計書を作成する。**実装には進まないこと。**

1. まず、AGENTS.md の詳細設計書トリガーに該当するか判定する。
   - 要件定義と計画書の間の飛躍が大きく、実装者（AI）が迷う複雑な機能
   - 複数の計画書にまたがって参照される共通基盤・アルゴリズム
   - 状態遷移・データフローが複雑で、文章より図表で固めるべきもの

   **該当しない場合は作成せず**、計画書（plan-new）の「実装アプローチ」で足りる旨を
   報告して停止する。
2. `docs/requirements/requirements.md` が approved であることを確認する（でなければ停止）。
3. `docs/designs/_TEMPLATE.md` を元に `docs/designs/DESIGN-NN-<GG>-<slug>.md` を作成し
   （GG＝要件定義書の frontmatter `generation:` の現行世代番号）、
   モジュール構成・データモデル・処理フローを、実装者（AI）が迷わないレベルまで具体化する。
   - 外部公開API・DBスキーマの定義は `docs/interfaces/` が正。重複記述せず参照する。
4. **「設計判断と理由」を必ず書く。** ADRトリガーに該当する判断は adr-new スキルで
   起票してID参照する。
5. 全体を提示し、承認（status: approved）を求めて停止する。
   承認後、対応する計画書の `design:` にIDを記載する。
